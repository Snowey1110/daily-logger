Inline editor, bookmark page system, and editable date/time

## Changelog

### Added
- **Inline Editor** — replaced the modal Composite Page Editor with an
  in-place editor that renders directly on the journal page. Text,
  sketch, and image editing happen on the page itself with an
  EditorSidebar on the left for tools and layer management.
- **EditorSidebar component** — new fixed left-side panel with layer
  tabs (text/sketch/images), sketch tools (draw, eraser with size
  preview, color picker, line width), image upload, layer reordering,
  and save/close actions. Portalled to document.body to avoid CSS
  transform clipping.
- **useInlineEditor hook** — custom hook encapsulating all inline
  editing state: canvas drawing (with eraser cursor), image
  drag/resize, text editing, and layer ordering.
- **Right-page editing** — click-to-switch between left and right pages
  while editing; active layer is preserved across side switches.
- **Editable date/time header** — the date and time fields in the
  inline editor header are now editable input fields that save back to
  the Excel file.
- **Bookmark page system** — Speech-to-Text and AI Report content is
  inserted as dedicated pages into the journal flow when the
  corresponding bookmark tab is active. Bookmark pages always appear on
  the right side of a spread, paired with the entry's journal text on
  the left.
- **Always-visible bookmark tabs** — Journal, Speech, and AI Report
  tabs are always shown (no longer conditionally hidden based on
  content).
- **Bookmark-aware navigation** — switching bookmark tabs rebuilds the
  spread list and navigates to the same entry in the new layout, keeping
  the user's reading position stable.

### Changed
- **Backend: editable date/time** — PATCH /api/entry and
  patch_journal_reader_entry() now accept optional date and time
  parameters to update columns 1-2 in the Excel sheet.
- **Spread builder (STT/AI mode)** — rewrote the bookmark branch to
  insert bookmark pages as regular PageItems and pair them with
  bookmark-aware logic: bookmark pages land on the right, entries with
  bookmarks get dedicated spreads, and entries whose journal was already
  consumed as a right page show their text alongside the bookmark.
- **Editor overlay positioning** — uses position: absolute within the
  book spread container instead of fragile position: fixed with
  getBoundingClientRect, eliminating zoom and scroll misalignment.
- **Book offset on edit** — uses transform: translateX(7rem) instead
  of padding-left to shift the book when the sidebar is open, avoiding
  layout reflow.
- **Page overlay always passed** — the Page component's overlay prop is
  no longer cleared during editing, preventing layout shifts from
  whitespace reflow.
- **Text layer background** — editor text area has an opaque background
  matching the book inner color, preventing rendered page text from
  showing through.

### Removed
- **CompositePageEditor** — removed the modal-based composite editor in
  favor of the inline editor.
- **Bypass page border** — removed the cross-page sketching feature
  after iterative testing showed it was unreliable.
- **displaySpread memo** — removed the intermediate useMemo that
  overrode spread content for bookmark viewing, replaced by direct
  spread rebuilding.
